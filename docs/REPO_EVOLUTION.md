# Repository Evolution

## Original State

The original project existed mainly as exploratory notebooks and saved model artifacts. That stage was useful for experimentation, but it had common prototype limitations:

- hard-coded local paths
- mixed experimental versions in the same notebook
- unclear distinction between extraction, cleaning, and modeling
- weak reproducibility
- limited explanation of labels and evidence

## Current Goal

The goal of this repository is to preserve the real work while presenting it in a clearer structure that is easier to explain in interviews and easier for collaborators to understand.

## Improvement Strategy

1. Separate business logic from notebook exploration.
2. Make the classification unit explicit.
3. Add annotation templates and codebook.
4. Start with an interpretable baseline.
5. Keep semantic modeling as a documented experiment.
6. Improve GitHub presentation with architecture, scope, and honest status.
